Description

OVERVIEW

Open LMS began a redesign of one of its most valuable plugins to create content, the centralized content management. We had the redesign in mind with the vision of becoming the preferred tool to help the administrators create and distribute content along the platform. The goal of this particular project was to identify any usability issues with this release release and determine the critical features to prioritize for the next iteration of the product.

BACKGROUND

Open LMS has the vision of efficiently supporting educational institutions and companies with a suite of technology and level of customer service. Learn more about Open LMS.
With this new plug-in that makes the work of content creators easier, we wanted to create a unique user experience by eliminating any look and feel issues, we wanted to remove any breakpoints in the workflow and cognitive load. We also wanted to improve the usability and increase the productivity of the tool.


I joined Open LMS as a UX/UI designer as one of the 2 designers in the company. I support design across every aspect of our business and I am responsible for the UX and UI across key parts of the product.


I've grown tremendously in the company and have some key achievements that I am very proud of.

  • I implemented user research and usability testings along the new tools. No usability tests were conducted before. Since we establish a design team, we have been actively working towards conducting UX research and usability testing on all projects.

    Establish a design kit This has helped to maintain consistency in the look and feel across different parts of the platform.

The Process

DESIGN PROCESS

Our process is based on Design Thinking.

1. Empathize : User Persona, journey map, empathy map.
2. Define : Research Analyze, affinity map, defined the user and creating the problem statement.
3. Ideate : User Flow, low-fidelity wireframe.
4. Build : Wireframes, visual brading, protyping.
5. Test : Run an usability test.

EMPATHIZE

UNDERSTANDING THE PROBLEM

The Centralized Content Manager tool had been implemented based on Moodle core tools. The plug-in had been created without any usability testing and had little consideration for the technical product limitation on the scope of work.


I conducted research interviews with our primary users (The onboarding team) to uncover any pain points that they were experiencing with the previous release. I also asked our users in the community, to complete a set of different tasks. Some questions I asked to discover any break point in the workflow I have listed below


  • How did you find the experience of Creating a replica?
  • Did you notice where to start using the CCM?
  • What was your overall impression of Creating a Master resource?
  • What was the best/worst thing about Creating a replica?
GATHERING INSIGHTS

After collecting the recordings from the user interviews, I conducted an affinity mapping with my teammate to synthesize the pain point identified. We also create a user journey to see the workflow and find any breakpoints.


We used the Severity Ratings for Usability Problems to inform any usability issues in order of priority. The framework helps to identify the severity score of a usability issue based on these variables

Task criticality x impact x frequency = severity

Task criticality How important is the task to the user?

Impact How much of an impact does this issue have on the user's task?

Frequency How many times does this come up out of total participants?

DEFINITION

Based on the user interviews conducted we found the following key issues:

  • 29% of users were frustrated that was hard to find the step to create a master
  • 29% of users found that it was easy to miss field inputs due to a lack of visual hierarchy
  • 14% of users found it hard to access the course since the link was not easy to find

WIREFRAMES

Based on the problems identified. We created wireframes from the solution by coming up with some potential solutions.

  • Build simple in-use interface: Grouping the related content and minimizing the steps for each creation of masters and replicas
  • Visual Hierarchy: Establish a visual hierarchy and group related elements. Create a standard interface patron to be easy to understand by future users.
  • Structure the content: Organised the information grouping and by priorization
  • User Center Design: Create a visual reference for steps to create a master and a replica, organised the content and element by relevancy.

I quickly mocked up some basic wireframes to gather feedback from Product, Engineering, and the users on the overall tool.

VALIDATING THE DESIGNS

We conducted usability testing with our primary users to validate whether the new designs would solve the problems. I wrote a scenario asking the user to do some tasks. Creating an announcement, creating a master and then a replica in the courses, finalizing with propagating the replicas.


This was a remote usability testing session in which I could observe the heat maps and how the users interact with the tool. They made some missions and then answered some questions. Seeing that it was easier to create one task, with better navigation, without so much text.


DEVELOPING THE DESIGNS

I created high-fidelity mockups in Adobe XD and made a prototype and walk-through with the designers. Then was the creation of the user stories tickets to start the development of the tool.


I worked very closely with the Front End team to spec out any missing interactions that were not covered in the prototypes. worked close reviewing each ticket to ensure it was aligned with the designs.

RESULTS

Along with this redesign we created tutorials online, so people would have a clear vision of what they can do with the tool. After the redesign, we got many compliments for the interface.


ACCESSIBILITY

We are very committed to accessibility and in the later releases, we updated the AX for the plugins. We added Keyboard navigation, Aria attributes unique attributes, interactive elements, focusable, alternative text, and color contrast. We continue with the improvement to aid people with different types of disabilities.
